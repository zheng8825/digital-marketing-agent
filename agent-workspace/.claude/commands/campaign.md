The marketer wants a campaign plan. Request: **$ARGUMENTS**

1. Read `knowledge/01-brand-and-products.md`, `knowledge/02-channels-and-partners.md`,
   `knowledge/04-calendar-and-moments.md`, `knowledge/strategy/00-brand-awareness-strategy.md`, and
   `knowledge/05-campaign-learnings.md` (apply past learnings). State any assumptions for missing inputs
   (budget, exact dates, hero SKU, primary KPI) and proceed.
2. Produce the plan with these sections:
   - **Snapshot** — campaign name, line/SKU, moment, dates (with a runway check), primary KPI + 2 supporting.
   - **Audience** — which MY segment(s) (students / young pros & creators / gamers / SOHO / family-gifters),
     and the insight you're playing to.
   - **Big idea / message + offer** — one line, plus the message wedge per sub-brand involved.
   - **Channel plan** — what runs where (Meta, Google Search/PMax/YT, TikTok, KOL, owned social, SEO/site,
     email, retailer co-marketing), each with objective + rough **budget split** (% of total) + key creative.
     Flag overlaps with channels already running and who to align with.
   - **Creative needs** — list of assets (formats, sizes, languages EN/BM/ZH, copy direction).
   - **Ad copy** — 3 headline + primary-text variants per main channel, in EN / BM / ZH as the audience needs.
   - **KOL plan** — pull from `knowledge/03-kol-list.md`; tiers, # of creators, deliverables, brief outline.
   - **Tracking** — the UTM tags to use (consistent with `outputs/analytics/utm-convention.md`), which GA4
     key events to watch, what the dashboard should show.
   - **Timeline** — week-by-week: brief → creative → KOL booking → ad build → live → optimise → wrap-up.
   - **Risks / dependencies / approvals needed.**
3. Save to `outputs/campaigns/<YYYY-MM>_<slug>/plan.md` (create the folder). Tell her the path. Offer to also
   draft the KOL briefs and the ad-account build-sheet as separate files in that folder.
4. After it runs (later), remind her to add a wrap-up entry to `knowledge/05-campaign-learnings.md`.
